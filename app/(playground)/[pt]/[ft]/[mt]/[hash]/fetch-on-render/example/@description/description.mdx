# Fetch-on-Render

Fetch-on-Render パターンでは、コンポーネントのレンダリング後にデータフェッチが行われます。

## Fetch-on-Render の動作

Fetch-on-Render では、コンポーネントがレンダリングされた後にデータのフェッチが行われます。これは通常、**useEffect** フック内で行われます。コンポーネントがマウントされると（つまり画面に表示されると）、**useEffect** 内のデータフェッチ処理がトリガーされ、データの取得が始まります。

## メリット

・**実装のシンプルさ:** Fetch-on-Render は直感的で、理解しやすい。初学者にとっても実装が容易です。

・**コンポーネントの分離:** データ取得と表示のロジックがコンポーネント内で分離されるため、管理がしやすいです。

## デメリット

・**ウォーターフォール問題:** データの依存関係により、一つのデータがフェッチされるのを次のデータが待つ「ウォーターフォール」が発生することがあります。これにより、ページのロードが遅くなる可能性があります。

・**レンダリングの遅延:** コンポーネントがマウントされてからデータフェッチが始まるため、ユーザーはデータがロードされるまで待たなければなりません。

## 運用ケース

Fetch-on-Render は、データの依存関係が少なく、シンプルなデータ取得が必要な場合に適しています。例えば、特定の API からの基本的なデータ取得や、ページに大きな影響を与えない軽量なデータ取得などが挙げられます。また、データ取得の複雑さを避けたい初学者や、小規模なプロジェクトにも向いています。
